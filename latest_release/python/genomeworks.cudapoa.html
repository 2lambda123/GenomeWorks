CUDAPOA binding module.

class genomeworks.cudapoa.cudapoa.CudaPoaBatch(max_sequences_per_poa, max_sequence_size, max_gpu_mem, output_type=u'consensus', band_mode=u'static_band', device_id=0, stream=None, gap_score=-8, mismatch_score=-6, match_score=8, alignment_band_width=256, max_consensus_size=None, max_nodes_per_graph=None, *args, **kwargs)¶

Bases: object

Python API for CUDA-accelerated partial order alignment algorithm.

add_poa_group(self, poa)¶

Set the POA groups to run alignment on.

Args:
poasList of POA groups. Each group is a list of

sequences. e.g. [[“ACTG”, “ATCG”], <— Group 1 [“GCTA”, “GACT”, “ACGTC”]] <— Group 2 Throws exception if error is encountered while adding POA groups.

batch_id¶

Get the batch ID of the cudapoa Batch object.

Returns:

Batch ID.

generate_poa(self)¶

Run asynchronous partial order alignment on all POA groups in batch.

get_consensus(self)¶

Get the consensus for each POA group.

Returns:

A tuple where - first element is list with consensus string for each group - second element is a list of per base coverages for each consensus - third element is status of consensus generation for each group

get_graphs(self)¶

Get the POA graph for each POA group.

Returns:

A tuple where - first element is a networkx graph for each POA group - second element is status of MSA generation for each group

get_msa(self)¶

Get the multi-sequence alignment for each POA group.

Returns:

A tuple where - first element is a list with MSAs for each POA group, with each MSA represented as a list of alignments. - second element is status of MSA generation for each group

reset(self)¶

Reset the batch object. Involves deleting all windows previously assigned to batch object.

total_poas¶

Get total number of POA groups added to batch.

Returns:

Number of POA groups added to batch.
